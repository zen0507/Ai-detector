# Falsum: AI Financial Fraud Detector

**Falsum** is an advanced, high-performance platform designed for the automated detection and forensic analysis of fraud in financial documents. By leveraging cutting-edge Machine Learning (ML), Computer Vision, and Statistical Analysis, the system identifies digital manipulations, suspicious patterns, and data inconsistencies across various financial instruments like invoices, receipts, and bank statements. The platform provides a multi-role environment that bridges the gap between automated AI scanning and human forensic verification.

---

## 👥 User Modules

The system is organized into three distinct, specialized roles, each serving a critical function in the fraud detection lifecycle:

### 1. **Admin Module**

The Admin acts as the central overseer of the entire platform. This module provides tools for:

- **User & Role Management**: Controlling access, managing accounts, and upgrading standard users or auditors to the "Forensic Expert" role.
- **System Monitoring**: Overseeing the health of the AI engine and tracking overall system performance.
- **Global Visibility**: Accessing high-level analytics and reports across all scanned documents and expert decisions.

### 2. **Forensic Expert Module**

The Forensic Expert serves as the final decision-maker for suspicious cases. This module includes:

- **Expert Dashboard**: A specialized interface for managing the manual review queue.
- **Deep-Dive Analysis**: Detailed views of AI-flagged indicators, such as image forensics (ELA) and statistical anomalies (Benford's Law).
- **Human Verdict**: The ability to provide the final "Approved" or "Rejected" status, along with forensic notes and evidence-based justifications.

### 3. **Auditor Module**

The Auditor is the primary "front-line" user responsible for data ingestion. Key features include:

- **Evidence Upload**: Drag-and-drop interfaces for uploading documents in bulk or individually.
- **Real-time Risk Tracking**: Immediate visibility into the automated risk scores (Low, Medium, High) generated by the AI.
- **Document History**: A personal repository of all uploaded items and their associated analysis results.

---

## ✨ Platform Features & Capabilities

The Falsum platform offers a comprehensive suite of tools for proactive fraud prevention. It features an **Automated AI Scanner** that processes multiple document types (Invoices, Receipts, Bank Statements, Purchase Orders) using a multi-engine OCR approach (EasyOCR and Tesseract). Beyond simple text extraction, the system performs **Image Forensics** using Error Level Analysis (ELA) to detect pixel-level manipulations. It further integrates **Statistical Forensics** via Benford's Law analysis to identify artificial digit distributions.

The platform provides **Real-time Risk Assessment**, categorizing documents into risk levels based on confidence scores. A **Forensic Queue** streamlines the workload for experts, ensuring high-priority suspicious items are reviewed promptly. **Dynamic Analytics Dashboards** provide visual representations of fraud trends, expert efficiency, and document distributions using interactive charts. With **Role-based Access Control (RBAC)** and a secure **Audit Trail**, the platform ensures every action, from initial upload to final verdict, is tracked and verifiable, making it an essential tool for maintaining corporate financial integrity.

## 📊 Database Schema & Tables

The core data architecture is built around the following relational entities in the PostgreSQL database:

| Table                | Description                                                                                                  |
| :------------------- | :----------------------------------------------------------------------------------------------------------- |
| **`User`**           | Default Django user model for authentication.                                                                |
| **`UserProfile`**    | Extends the User model with specific forensic roles (**Auditor**, **Forensic Expert**, **Admin**).           |
| **`EvidenceItem`**   | Stores the uploaded raw files, original filenames, upload timestamps, and soft-deletion flags.               |
| **`AutomatedScan`**  | Contains AI analysis results: document type, risk level, confidence scores, and raw fraud indicators (JSON). |
| **`ForensicReview`** | Stores the manual human verdict (Approved/Rejected/Pending), expert notes, and session metadata.             |

## 🔄 System Flow

1. **Upload**: An **Auditor** uploads a document via the React UI.
2. **Scan**: Django persists the file and triggers a request to the **FastAPI AI Engine**.
3. **Analyze**: The AI Engine performs OCR, Image Forensics, and Statistical analysis.
4. **Identify**: Anomalies are flagged (e.g., "Non-standard Font", "Benford Outlier", "ELA Anomaly").
5. **Verdict**:
   - Low-risk items are marked as processed.
   - High-risk/Suspicious items are queued for a **Forensic Expert**.
6. **Review**: The expert analyzes the flags and provides the final human verdict.
7. **Report**: Aggregated data is visualized on **Admin Dashboards** and can be exported as PDF reports.
