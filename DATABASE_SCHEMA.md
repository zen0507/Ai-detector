# Database Schema Documentation

## 1. Table: `detector_evidenceitem`
**Core Entity**. Stores the immutable "Case File" and chain-of-custody data. This is the parent record for any analysis or review.

| Column Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | BigInteger | PK | Unique ID for the evidence item. |
| `user_id` | Integer | FK | ID of the user who uploaded the document (links to `auth_user.id`). |
| `uploaded_file` | String | | Relative path to the stored file on disk (e.g., `documents/2025/12/27/image.png`). |
| `filename` | String | | Original name of the uploaded file. |
| `uploaded_at` | DateTime | | Timestamp of when the upload occurred (Auto-added). |
| `is_deleted` | Boolean | | Soft-delete flag (records are hidden, not wiped from DB). |
| `deleted_at` | DateTime | | When the record was deleted. |
| `deleted_by_id` | Integer | FK | Who deleted the record (links to `auth_user.id`). |
| `deletion_reason` | Text | | Reason provided for deleting the record. |

---

## 2. Table: `detector_automatedscan`
**AI Intelligence**. Stores the results of the automated fraud detection process. Linked 1-to-1 with an Evidence Item.

| Column Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | BigInteger | PK | Unique ID for the scan result. |
| `evidence_id` | BigInteger | FK, UQ | Link to the parent Evidence Item. One-to-One relationship. |
| `ai_analysis_id` | String | Index | Unique Hash ID generated by the FastAPI backend for this specific scan. |
| `document_type` | String | | Detected type (e.g., 'invoice', 'receipt', 'unknown'). Default: 'unknown'. |
| `detection_confidence` | Float | | System's confidence in the document type (0.0 - 1.0). |
| `final_score` | Float | | **Fraud Probability Score** (0.0 - 1.0). Higher means more likely fake. |
| `risk_level` | String | | Classification: `low_risk`, `medium_risk`, `high_risk`, `non_financial`. |
| `fraud_indicators` | JSON | | List of detected anomalies (e.g., text alignment issues, pixel grids). |
| `component_scores` | JSON | | Breakdown of scores from different AI models. |
| `extracted_text` | JSON | | Raw OCR results and text content. |
| `recommendations` | JSON | | Dynamic suggestions for risk mitigation based on findings. |
| `is_processed` | Boolean | | `True` if analysis is complete; `False` if pending or failed. |
| `processing_error` | Text | | Stores error messages if the AI backend fails. |

---

## 3. Table: `detector_forensicreview`
**Human Verdict**. Stores the decision made by a human expert/admin regarding the evidence. Linked 1-to-1 with an Evidence Item.

| Column Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | BigInteger | PK | Unique ID for the review record. |
| `evidence_id` | BigInteger | FK, UQ | Link to the parent Evidence Item. One-to-One relationship. |
| `status` | String | | Status of review: `not_requested`, `pending`, `approved` (verified), `rejected` (fraud). |
| `reviewer_id` | Integer | FK | ID of the designated expert/staff member (links to `auth_user.id`). Nullable. |
| `reviewed_at` | DateTime | | Timestamp of the last review action. Auto-updates on save. |
| `notes` | Text | | **Expert Notes**. Custom comments or justification for the verdict entered by the expert. |

---

## 4. Table: `auth_user`
Standard Django authentication table storing account information for officers and admins.

| Column Name | Type | Key | Description |
| :--- | :--- | :--- | :--- |
| `id` | Integer | PK | Unique User ID. |
| `username` | String | | Unique username for login. |
| `first_name` | String | | User's first name. |
| `last_name` | String | | User's last name. |
| `email` | String | | User's email address. |
| `password` | String | | Hashed password string. |
| `is_staff` | Boolean | | Designates whether the user can log into this admin site/perform reviews. |
| `is_active` | Boolean | | Designates whether this user should be treated as active. |
| `is_superuser` | Boolean | | Designates that this user has all permissions without explicitly assigning them. |
| `last_login` | DateTime | | Timestamp of the last successful login. |
| `date_joined` | DateTime | | Timestamp of account creation. |
